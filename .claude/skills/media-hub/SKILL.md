# Media Hub - Google Drive Asset Management

Central media relay for all generated and personal assets. Lives on a shared drive in the owner's Google Workspace ({{OWNER_EMAIL}}).

## Setup (first use)

During onboarding Phase 4 (Google Workspace connection), create a shared drive named `Media Hub` plus the folder structure below, then fill the IDs into this table. Until the IDs are filled, this skill should offer to create the structure rather than upload blind.

## Config

**Shared Drive ID:** `{{MEDIA_HUB_DRIVE_ID}}`

## Operations

### Upload a local file
```bash
gws drive +upload "<LOCAL_PATH>" --parent "<FOLDER_ID>" --name "<FILENAME>" --format json
```

### Create a Google Doc
```bash
gws drive files create \
  --params '{"supportsAllDrives": true, "fields": "id,name,webViewLink"}' \
  --json '{"name": "<TITLE>", "parents": ["{{FOLDER_GOOGLE_DOCS}}"], "mimeType": "application/vnd.google-apps.document"}' \
  --format json
```

### Create a Google Sheet
```bash
gws drive files create \
  --params '{"supportsAllDrives": true, "fields": "id,name,webViewLink"}' \
  --json '{"name": "<TITLE>", "parents": ["{{FOLDER_SPREADSHEETS}}"], "mimeType": "application/vnd.google-apps.spreadsheet"}' \
  --format json
```

### List files in a folder
```bash
gws drive files list \
  --params '{"q": "\"<FOLDER_ID>\" in parents", "supportsAllDrives": true, "includeItemsFromAllDrives": true, "fields": "files(id,name,mimeType,modifiedTime)"}' \
  --format json
```

### Copy a file into the hub
```bash
gws drive files copy \
  --params '{"fileId": "<SOURCE_FILE_ID>", "supportsAllDrives": true, "fields": "id,name"}' \
  --json '{"parents": ["<TARGET_FOLDER_ID>"]}' \
  --format json
```

## Auto-Routing Logic

1. **Image generated by AI** → `Generated/Portraits`
2. **Video generated by AI** → `Generated/Videos`
3. **NotebookLM artifact** → `Generated/NotebookLM`
4. **Audio file** → `Audio`
5. **Google Doc created** → `Documents/Google Docs`
6. **Google Sheet created** → `Documents/Spreadsheets`
7. **Social visual** → `Social`
8. **Blog/newsletter image** → `Content Assets`
9. **The owner uploads a photo** → `Personal Photos/Social Fuel`
10. **AI training reference** → `Training Data`

## After Upload

Always confirm:
```
SAVED to Media Hub:
- File: [filename]
- Folder: [folder path]
- Link: [webViewLink if available]
```

## When The Owner Mentions Adding Photos

If the owner says "I uploaded some photos" or "I added pictures to the drive", list the contents of `Personal Photos/Social Fuel` to see what's new. These photos become available for social post visuals.
