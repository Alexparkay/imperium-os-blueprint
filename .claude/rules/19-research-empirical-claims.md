# MANDATORY: Research Before Any Empirical Claim

## The rule

Before stating ANY claim about external reality, you MUST verify via web search in the current session. No exceptions. No "I'll trust my training data on this one." No "the skill file says X." No "I'm pretty sure."

## What counts as an empirical claim (search required)

- Platform algorithm behavior (Instagram, LinkedIn, YouTube, TikTok, X, any social platform)
- Engagement benchmarks, format performance, content type comparisons
- Audience demographics, income data, consumption patterns
- Tool capabilities, API limits, pricing, feature availability
- Market trends, industry data, competitive intelligence
- News, events, current state of any company, product, or person
- Statistics of any kind
- "What works" / "what performs" / "what's trending" claims
- Any claim where reality could have moved in the last 30 days

## What does NOT require search

- Internal repo state (use Read, Grep, Bash)
- The owner's own memory files, rules, skills, conversation history
- Pure reasoning, frameworks, strategic logic that doesn't depend on current data
- Math, code, syntax, language mechanics
- Conversational responses with no factual claims

## Freshness rules for internal files

- **Any skill file, memory file, or rule older than 14 days** must be sanity-checked against current reality before citing its empirical claims
- **Older than 30 days** = must web search before treating as current truth
- If a skill file says "98% of X do Y" and that claim is empirical, verify it even if the skill file is only 10 days old
- Internal docs are snapshots, not ground truth

## How to apply

1. Before writing any claim, ask: "is this empirical?"
2. If yes, ask: "did I verify this in the current session via tool output?"
3. If no to verification, STOP. Run WebSearch / WebFetch / Grep before continuing
4. Cite the source inline or in a Sources section

## What this prevents

- Pattern-matching from training data when reality has moved
- Citing stale skill files as authoritative
- Giving confident-sounding answers with no underlying verification
- The "I was wrong, let me search, I was right" pattern that costs 3x the conversation
