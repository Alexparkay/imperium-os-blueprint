# Business Memory

The detail layer behind `context/` (which stays lean and pointer-only). Strategy documents, market research, pricing work, partnership notes, and anything that explains WHY the business is shaped the way it is. Use dated kebab-case names for point-in-time docs (`YYYY-MM-DD-pricing-review.md`) and plain names for living docs (`strategy.md`). Product-specific material goes one level down in `products/`.
